# GENERIC ITERATIVE REFINEMENT METHODOLOGY

## Overview
This methodology enables deep, iterative refinement of multiple deliverables from multiple source documents through systematic reading, analysis, creation, and cross-pollination cycles.

## Parameters
- **N** = Number of source documents
- **M** = Number of deliverables to create
- **P** = Number of full passes (recommended: 2 minimum)

## Process Structure

### Setup Phase
1. Identify all N source documents
2. Define all M deliverables with clear descriptions
3. Create TODO tracking document listing all operations

### For Each Pass (1 to P):

#### For Each Deliverable (1 to M):
1. **Sequential Document Reading**
   - For each source document (1 to N):
     - Read document with focus on current deliverable
     - Write thoughts/notes specific to current deliverable
     - Perform web searches for context/validation

2. **Deliverable Creation/Refinement**
   - Create or refine the current deliverable
   - Incorporate insights from all N documents
   - Perform additional validation searches

3. **Cross-Refinement Phase** (if not on deliverable 1)
   - Revisit previous deliverables (M-1, M-2, ..., 1)
   - Apply insights gained from current deliverable
   - Perform targeted searches for each revision
   - Update each previous deliverable

### Progression Notes
- **First Pass**: Focus on creation and initial coherence
- **Subsequent Passes**: Focus on refinement, depth, consistency, and quality improvement
- **Cross-Refinement**: Each deliverable informs and improves all previous deliverables
- **Search Integration**: Searches validate assumptions, fill gaps, and add authoritative context

### Quality Targets
- Each pass should achieve minimum 20% improvement
- Final deliverables should be internally consistent
- All deliverables should be mutually reinforcing
- External sources should validate and contextualize approaches

## Adaptation Guidelines
- Scale N (documents) and M (deliverables) as needed
- Adjust P (passes) based on complexity and quality requirements
- Add intermediate review gates for complex projects
- Integrate domain-specific validation steps
- Track completion in structured TODO format

## Success Criteria
- All TODO items completed
- Each deliverable meets defined scope
- Cross-references are accurate and consistent
- External validation supports key claims
- Improvement targets achieved across all passes
